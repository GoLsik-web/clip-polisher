Автомонтаж вертикальных клипов из Twitch-клипов.

## РОЛЬ И СТИЛЬ

Опытный Python-инженер (видео + десктоп). Пользователь не программист, код не читает.
Инкрементально, маленькими проверяемыми кусками: каждый модуль сначала в `core/` +
тест на реальном файле, показать результат, потом UI. По важным развилкам — спрашивать,
по мелочам — разумные дефолты в README. Комментарии к нетривиальному — по-русски.

## ЧТО СТРОИМ

Windows-приложение: вход — горизонтальный Twitch-клип (файл ИЛИ ссылка), выход —
вертикальный клип 9:16 (TikTok/Shorts), готовый к постингу: раскладка (вебка+геймплей),
анимированные субтитры, брендинг, нормализованный звук.
Этап 1 из трёх (НЕ сейчас): Этап 2 = метки вживую + склейка отрезков; Этап 3 = автопоиск ИИ.

## СТЕК (зафиксировано)

Python 3.11+ · FFmpeg через subprocess, GPU h264_nvenc · faster-whisper large-v3, GPU
(CUDA), word_timestamps=True · субтитры .ass (libass) · yt-dlp · PySide6 (Qt) ·
PyInstaller → .exe.

## АРХИТЕКТУРА (жёстко)

- `core/` — вся логика, БЕЗ импорта Qt. Чистые функции/классы (CLI/скрипт/облако). «Мозг».
- `ui/` — тонкая оболочка PySide6, только собирает настройки. core НИКОГДА не импортирует ui.
- FORWARD-COMPAT: render/pipeline собирает клип из СПИСКА сегментов `[(start,end),...]`
  (Этап 1 — один сегмент), чтобы Этап 2 добавил склейку до 4 отрезков без переделки.

## СТРУКТУРА

```
clip-polisher/
```

## ПАЙПЛАЙН (pipeline.py)

1 Ingest → 2 Trim (список сегментов, сейчас один) → 3 Transcribe → 4 Profanity
(если вкл: *** + интервалы бипа) → 5 Captions (.ass) → 6 Layout (9:16) → 7 Branding →
8 Audio (loudnorm −14 LUFS + бипы) → 9 Render (h264_nvenc, разрешение/fps → mp4).

## МОДУЛИ

- **layout.py** — 4 пресета 9:16 (1080×1920). Зоны в долях исходного кадра. A(дефолт)
  вебка↑+геймплей↓; B геймплей на весь+вебка-кружок; C лицо на весь; D геймплей↑+вебка↓.
  Пресеты — JSON в presets/.
- **captions.py** — .ass из словных таймкодов. Шрифт, размер, цвет, обводка(толщина+цвет),
  фон-плашка(цвет+прозрачность), позиция. 4 анимации (data-driven): Pop(\t fscx/fscy),
  Fade(\fad), Slide-up(\move), Karaoke(\k). Каждая — отдельный генератор ass-тегов.
- **profanity.py** — русский словарь со словоформами; переключатель; при вкл: *** в тексте
  + бип 1 кГц поверх интервала слова.
- **branding.py** — ник + иконка платформы (Twitch/YouTube/Kick), позиция/размер настраиваемые.
- **audio.py** — loudnorm (I=-14, TP=-1.5) + бипы. Музыка — не сейчас.
- **render.py** — 1080×1920 (опция 720×1280), fps 30/60, h264_nvenc, AAC; имя/папка;
  принимает СПИСОК сегментов.
- **preview.py** — стоп-кадр с раскладкой/субтитрами ИЛИ картинка пользователя. Живого видео нет.
- **Режим**: «по одному / пачкой».

## ИНТЕРФЕЙС (PySide6) — эталон: docs/ui-reference.html

Воспроизвести компоновку/флоу/цвета/поведение/ощущение анимаций средствами Qt
(QSS + QPropertyAnimation + QGraphicsView). Пиксельная идентичность не нужна — важны
структура, логика и «вкусность».

### Палитра (из эталона)
- Тёмная (дефолт): bg #08080d, panel #111019, panel2 #191826, line #2a2740,
  text #f9f8ff, muted #c2bde0, accent #7c5cff.
- Светлая (опция): bg #edeaf6, panel #fff, panel2 #f3f1fb, line #e2ddf1,
  text #0d0b14, muted #443f58, accent #6a3cff.
- Цвета зон (постоянные во всём приложении): вебка #ff5aa0, геймплей #5590ff,
  субтитры #f4b13f, ник #57d081, платформа #ff8149.
- Цвета окон 1–6: #7c5cff, #5590ff, #37c9c2, #57d081, #f0a93a, #ff6ba6.

### Компоновка (режим 1)
Топбар (лого + «По одному/Пачкой» + тема ◑). Рабочая зона: [РЕЙКА РЕЖИМОВ слева] →
[МАСТЕР настроек] → [ПРЕВЬЮ справа].
- **Рейка режимов** — вертикальная панель: 1 Twitch-клипы (рабочий), 2 Метки через
  бота (Этап 2, заглушка), 3 Автопоиск ИИ (Этап 3, заглушка). 2/3 — пустые каркасы.
- **Мастер** — пошаговый аккордеон: 1 Вход → 2 Раскладка → 3 Обрезка → 4 Субтитры →
  5 Брендинг → 6 Экспорт. Раскрыт ровно ОДИН шаг. «Далее» → текущий плавно сворачивается,
  номер → галочка, раскрывается следующий; «Назад»; по завершённым можно кликнуть.
  Аккордеон на QPropertyAnimation (maximumHeight + opacity), автоскролл к активному.
- **Превью справа (sticky)** — ОДНОВРЕМЕННО две версии: «Альбомная» (16:9, исходный кадр)
  и «ТикТок» (9:16, итог). Обе с полупрозрачными ЦВЕТНЫМИ ЗОНАМИ + ЛЕГЕНДА. Кнопка
  «Изменить кадр». Под превью — ТАЙМЛАЙН обрезки со стоп-кадрами, ручками in/out,
  плейхедом, таймкодами.

### Холст-редактор зон
Зоны (Вебка/Геймплей/Субтитры/Ник/Платформа) — полупрозрачные цветные области поверх
кадра, постоянный цвет. Прямое манипулирование на «ТикТок»-версии: перетаскивание
(позиция), углы (масштаб), верхняя ручка (поворот). QGraphicsView/QGraphicsScene +
кастомные QGraphicsItem с ручками, snapping к краям/центру, живое обновление.

### Разноцветность окон 1–6
Каждому окну свой цвет (мягко): в номере шага, рамке/свечении активного шага, и в
КОНТРОЛАХ ЭТОГО окна (чипы, тумблеры, ползунки, «Далее», подсветка пресетов, ховер «i»).
В Qt нет CSS-переменных → шаблонизируемый QSS (подставлять цвет активного шага) или
dynamic property + repolish при смене шага.

### Фон (воспроизвести ощущение)
- Еле заметная ТЕКСТУРА — точечная сетка (мелкие точки, низкая непрозрачность).
- ОДНА размытая ФИОЛЕТОВАЯ СФЕРА (мягкое свечение):
  - гуляет ТОЛЬКО по боковым свободным полям (гуттерам), не под интерфейсом; размер
    адаптируется под ширину поля;
  - плавно летит вертикально (~110 px/с), уходит за край → появляется с рандомной
    стороны; движение time-based (по delta, clamp dt);
  - ПЕРЕКРАШИВАЕТСЯ в цвет активного окна ПЛАВНО (интерполяция).
- Реализация: фоновый QWidget с paintEvent (радиальный градиент + блёр), позиция/цвет
  по таймеру, ПОД контентом. Сферу рисовать в offscreen-пиксмап (не блёрить каждый кадр).

### Прочий UX
- «i»-помощь у КАЖДОГО контрола (переиспользуемый HelpIcon): ховер → короткое пояснение.
- Обрезка — таймлайн со стоп-кадрами и ручками, не числовое поле. Пробел — play/pause.
- Прогресс рендера — НЕ консоль, а in-UI загрузчик-«прикол»: весёлая анимация (хлопушка)
  + процент + подменяющиеся шутливые русские статусы, привязанные к реальным этапам.
  Неблокирующий (QThread), сворачиваемый.
- Тёмная тема по умолчанию, светлая — опция; высокий контраст в обеих.
- Настройки проекта сохраняются/загружаются (QSettings/JSON).

### Анимации — идеальная плавность (обязательно)
60 fps, без рывков. QPropertyAnimation с easing (OutCubic/OutQuint, НЕ Linear). Единые
тайминги (окно ~350–450 мс, мелочь ~150–250 мс). Анимации прерываемые/реверсивные
(меняем endValue на лету). Сворачивание+разворачивание — синхронно (QParallelAnimationGroup).
Сфера — строго time-based. Тяжёлое (Whisper/FFmpeg) — в QThread, чтобы анимации не
запинались. Кэшировать дорогое (сфера — offscreen-пиксмап). Уважать «уменьшить движение».

### Стабильность на мониторах
HiDPI (High-DPI awareness; иконки SVG/@2x). Только layout-менеджеры (QHBox/VBox/Grid/
QSplitter), без абсолютных координат. Превью держат 16:9 и 9:16; зоны в долях.
setMinimumSize; при нехватке — QScrollArea. QSplitter рейка/мастер/превью с сохранением.
Шрифты в pt. Стартовые размер/позиция окна (QSettings). Проверить 1366×768, 1920×1080,
2560×1440 при 100/125/150%.

### Объём дизайна
ТОЛЬКО функциональный прототип (Пасс 1): структура/флоу/цвета/анимации. Не доводить
графику до идеала — финальный визуал сделает художник ПОСЛЕ. Чистые точки для стилизации:
единый QSS-файл, косметика отдельно от логики.

## ДЕФОЛТЫ

Whisper large-v3, float16 (при OOM → medium); loudnorm −14 LUFS; экспорт 1080×1920 @30;
**мат-фильтр по умолчанию ВЫКЛЮЧЕН** (переключатель).

## ТИПИЧНЫЕ ГРАБЛИ

- faster-whisper GPU: нужны CUDA/cuDNN; частый тихий откат на CPU. ЯВНО логировать
  устройство; нет GPU → предупреждать. VRAM 8 ГБ: float16/int8_float16; OOM → medium.
- h264_nvenc есть не во всех сборках FFmpeg — проверять реальным энкодом, фолбэк libx264.
- Кириллица в .ass: шрифт с кириллицей в assets/fonts + fontsdir для libass.
- .ass: цвет &HAABBGGRR (BGR+инверт. альфа), тайминги H:MM:SS.cs, экранирование;
  в Format событий ОБЯЗАТЕЛЬНО поле Name.
- NVENC — чётные ширина/высота. A/V-синхрон после фильтров. yt-dlp: клип ≠ VOD, ошибки.
- Пути с пробелами/кириллицей. Qt: тяжёлое в QThread, прогресс FFmpeg через -progress.
- PyInstaller: ffmpeg/шрифты/иконки/(модель) через sys._MEIPASS; модель — качать при
  первом запуске или бандлить.

## КРИТЕРИИ ГОТОВНОСТИ

- Реальный Twitch-клип (файл и ссылка) → готовый вертикальный mp4.
- Субтитры точные/синхронные/стилизуемые; ≥4 анимации; кириллица ок.
- 4 пресета корректны; зоны на холсте цветные, манипулируемые.
- Мат-фильтр пикает в тексте и звуке. Ник+иконка на месте; громкость нормализована.
- Мастер: пошаговое раскрытие, разноцветность окон 1–6, контролы в цвет окна, сфера
  гуляет по полям и перекрашивается — ВСЁ идеально плавно (60 fps).
- Два превью (16:9 и 9:16) с зонами, таймлайн-обрезка со стоп-кадрами.
- Загрузчик-прикол вместо консоли; «по одному» и «пачкой».
- Верстка стабильна на разных разрешениях/масштабах; GPU реально используется.
- Собирается в .exe, запускается на чистой Windows.
