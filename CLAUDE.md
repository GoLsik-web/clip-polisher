## Что за проект

Десктоп-приложение под Windows: горизонтальный клип стрима (файл ИЛИ ссылка на
Twitch-клип) → вертикальный клип 9:16 (TikTok/Shorts), готовый к постингу:
раскладка (вебка + геймплей), анимированные субтитры, брендинг, нормализованный звук.

**Текущий этап: Этап 1** из трёх. (Этап 2 — склейка разнесённых отрезков; Этап 3 —
авто-поиск моментов ИИ. Сейчас их НЕ делаем.)

## Стек (зафиксирован, не менять без спроса)

- Python 3.11+ (на машине 3.12.10)
- FFmpeg через subprocess, GPU-кодек **h264_nvenc** (фолбэк libx264)
- faster-whisper, модель **large-v3**, на GPU (CUDA), `word_timestamps=True`
- Субтитры: генерация **.ass** (libass) — стили/обводка/фон/анимации
- yt-dlp — скачивание клипов по ссылке
- PySide6 (Qt) — интерфейс
- PyInstaller → .exe (без цифровой подписи)

## Железо

Windows 11, **RTX 3060 Ti (8 ГБ VRAM)**, драйвер 610.74. Обработка локально на GPU.
VRAM 8 ГБ: `compute_type="float16"` (или `int8_float16`); при OOM — фолбэк на `medium`.

## Главный архитектурный принцип

Жёсткое разделение слоёв:
- **`core/`** — вся логика, БЕЗ Qt. Чистые функции/классы, вызов из CLI/бэкенда.
- **`ui/`** — тонкая оболочка PySide6: собирает настройки, вызывает `core`.
- **`core` НИКОГДА не импортирует `ui`.**

Любую фичу: сначала `core` + тест на реальном файле через CLI, потом UI.

## Forward-compat (заложено под Этап 2)

Рендер собирает клип из **СПИСКА сегментов** `[(start, end), ...]`, а не из одного
отрезка. На Этапе 1 список всегда из одного элемента. Модель данных и код рендера
без переделки держат до 4 сегментов с настраиваемым стыком. UI склейки сейчас НЕ делаем.

## Принятые дефолты

- Разрешение экспорта: **1080×1920**, опция 720×1280. FPS: **30**, опция 60.
- Аудио: loudnorm EBU R128 **I=-14, TP=-1.5**; бип-тон **1 кГц** поверх мата.
- **Мат-фильтр по умолчанию ВЫКЛЮЧЕН** (переключатель в UI). [обновлено по новому ТЗ]
- Раскладка по умолчанию: **пресет A** (вебка сверху + геймплей снизу). Зоны — в долях кадра.
- Субтитры: 4 анимации (Pop / Fade / Slide-up / Karaoke), реестр расширяемый.
- Брендинг: ник + иконка платформы (Twitch/YouTube/Kick/нет), дефолт — уголок.
- Whisper: large-v3, float16, GPU; лог устройства обязателен.

## Типичные грабли (детали — в SPEC, раздел «ТИПИЧНЫЕ ГРАБЛИ»)

- Whisper GPU: тихий откат на CPU / «cublas/cudnn not found» → ЯВНО логировать устройство.
- h264_nvenc: есть не во всех сборках FFmpeg + завязан на версию драйвера → проверять
  реальным тест-энкодом, фолбэк libx264 с предупреждением.
- .ass: цвет `&HAABBGGRR` (BGR + инверт. альфа), тайминги `H:MM:SS.cs`.
- Кириллица в libass: класть шрифт с кириллицей в assets/fonts, явно указывать fontsdir.
- NVENC: только ЧЁТНЫЕ ширина/высота — все кропы/паддинги приводить к чётным.
- A/V-синхрон после фильтров; пути с пробелами/кириллицей; парсинг `-progress` для прогресс-бара.
- **loudnorm РЕСЕМПЛИТ до 192 кГц** → без `aresample=48000` ПОСЛЕ него AAC выходит 96 кГц,
  и файл «играет 2 сек и виснет»/не открывается в плеерах и соцсетях. Всегда возвращать 48 кГц.
- `-shortest` + бесконечный `sine` (бип): из-за задержки loudnorm режет видео раньше времени.
  Тон делать КОНЕЧНЫМ (`sine=...:duration=<клип+1>`), amix=first, без глобального `-shortest`.
- `-progress`/`out_time_ms` НЕ монотонен при видео+аудио (буфер loudnorm убегает) → прогресс-бар
  в UI держать монотонным (`_target/_display`, назад не откатывать).

## СВОБОДНАЯ КОМПОЗИЦИЯ (новая модель раскладки, по запросу пользователя)

## Итерация «выбор зон + фиксы багов»

- КРАШ ИСПРАВЛЕН: смена формата после загрузки клипа рушила приложение — причина
  `QThread: Destroyed while thread is still running` (потоки хранились в одном атрибуте и
  перезаписывались на ходу). Фикс: `_track()` держит ссылки в `self._threads` (удаляем по
  `finished`), превью с ДЕБАУНСОМ (`_preview_running/_preview_pending` — не плодим потоки),
  `closeEvent` ждёт потоки (`th.wait`). Проверено: 5 быстрых refresh → 1 поток; смена формата ок.
- Легенда → КНОПКИ ВЫБОРА зоны (эксклюзивно): чтобы редактировать зону, надо сначала нажать
  её кнопку; редактируется ТОЛЬКО выбранная (`ZoneBox.editable`, у остальных
  `WA_TransparentForMouseEvents` — клики проходят мимо, пунктирная приглушённая рамка, без ручек).
  Выбранная зона поднимается наверх (raise). Кнопки недоступны для зон, которых нет в текущем холсте.
- Холст ВПИСЫВАЕТСЯ (contain) в доступное место (`CanvasArea.refit`), а не растёт огромным в 9:16 —
  легенда-кнопки и таймлайн всегда видны снизу. Редактор заполняет высоту колонки.
- Бургер УПРОЩЁН до 5 слоёв (2 булки белые + 3 начинки = 3 режима, цвета окон STEP_COLORS[0..2]);
  морф: начинки → 3 кнопки меню (цвета совпадают), полоски РАСТВОРЯЮТСЯ (alpha-fade) по мере
  проявления контента — без грязного наложения.
- Полный сценарий вход→редактирование→смена формата→рендер прогнан на реальном клипе — без краша,
  рендер из UI-конфига даёт mp4.

## Качество распознавания субтитров (улучшено)

- «СЛЫШИТ ЛУЧШЕ»: перед распознаванием — очистка речи FFmpeg-фильтрами
  (`_enhance_speech`): highpass 95 + `afftdn` (FFT-денойз фоновой музыки/шума) + lowpass 7800 +
  `dynaudnorm` (динамич. нормализация — поднимает тихий голос), моно 16 кГц. `denoise=True` по умолч.
  Видимый эффект: «скажу кушай» → «Слушай»; ловит мат/имена, которые терялись.
- «ПОНИМАЕТ ЛУЧШЕ»: beam_size=5, best_of=5, patience=2, temperature-фолбэк,
  condition_on_previous_text=True, no_repeat_ngram_size=3 + repetition_penalty (меньше повторов),
  log_prob_threshold=-1.2 (не выбрасываем неуверенные), hallucination_silence_threshold=2.0
  (убирает «странные» слова в тишине). VAD мягче (threshold 0.5→0.35, speech_pad 250) — ловит тихое.
- ТАЙМИНГИ: словные тайминги санитайзятся (end>start); в captions `_word_end` держит слово
  непрерывно до следующего, но в долгой паузе (>0.7с) прячет после короткого удержания
  (не зависает в тишине). Параметры transcribe вынесены (initial_prompt/denoise/vad).
